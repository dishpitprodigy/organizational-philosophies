# TDA Performance Evidence Ledger

This template prevents annual review from becoming memory reconstruction. It connects accountabilities, measures, changed conditions, development actions, employee reflection, manager observation, and next decisions into one evidence record.

The ledger is not a surveillance file. It is a reconciliation tool. It should preserve useful evidence without turning every developmental conversation into a disciplinary artifact.

## Use

1. Define accountabilities and expected evidence early.
2. Update the ledger when meaningful work, feedback, scope changes, or development reps occur.
3. Label evidence types.
4. Reconcile mid-cycle and end-cycle.
5. Feed the result into the next Personal Development Plan, 70-20-10 plan, assignment decision, promotion decision, or support change.

## Evidence Labels

| Label | Meaning |
|---|---|
| Observed behavior | Something directly seen in work, meetings, incidents, artifacts, delivery, or collaboration. |
| Outcome measure | A metric, target, service level, completion record, quality result, or other measured result. |
| Work artifact | A runbook, document, design, automation, ticket, test, project output, or other created object. |
| Self report | The employee's account of what happened, what they intended, or what they learned. |
| Manager interpretation | The manager's read on what the evidence means. |
| Peer or stakeholder input | Feedback from someone affected by, or close to, the work. |
| Assumption | A claim that still needs confirmation. |
